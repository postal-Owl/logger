Изначально при написании совершенно другого кода у меня появилась необходимость отслеживать как часто у меня вызываются конструкторы создания/копирования и т.д.
В итоге это переросло в небольшую библиотеку logger.

Для использования logger в своем проекте создаем Объект класса Logger (Для удобства можно создать в глобальном пространстве), например так:

    include <...>
    ...
    Logger logger("log.txt");  // log.txt - выходной файл куда будут записываться логи событий
    
    ... дальше ваш код

Также можно воспользоваться паттерном observer для отслеживания событий и создать простенький класс/структуру типа такой:

    struct MyEvent {
    //public:
    MyEvent(std::string msg) {
        if (logger.is_active())
            logger << msg;
    }
    void operator()(std::string msg) {
        if (logger.is_active())
            logger << msg;
    }
    ~MyEvent() {};

    };

На сколько мне известно это считается хорошей практикой (кажется это называется делегирование задач или разделение ответственности), т.к. при написании функций/методов/классов не стоит лишний раз расширять их функционал тем, чем они не должны заниматься.
Классам, функциям и методам помимо своего основного функционала лучше оставить возможность создавать только такие вот Events. И уже прочий функционал, в частности логирование, реализовать как реакцию на события. 

После создания объекта класса Logger в глобальном пространстве его можно будет использовать в любой части кода.
Примеры использования:
    
    logger.log("Был вызван конструктор");            // например так
    logger << "Был вызван конструктор копирования";  // или так (по сути это одно и тоже)

    MyEvent event("Событие создано");  // если использовать выше описанное событие
    event("фывэ");                     // ну вы я надеюсь поняли..
